## 介绍

> 看看_同一个_ Medium.com 克隆应用，如何用不同的[前端](https://codebase.show/projects/realworld?category=frontend)和[后端](https://codebase.show/projects/realworld?category=backend)实现。没错，你可以自由组合它们，因为**它们都遵循同一套 [API 规范](/docs/simple-world/endpoints)** 😮😎

大多数 “todo” 示例都能快速展示一个框架的能力，但通常无法传达真正构建_真实_应用所需要的知识和视角。

**RealWorld** 通过允许你选择任意前端（React、Angular 等）和任意后端（Node、Django 等）来解决这个问题。

## 了解更多

- 每个教程都基于同一套 [API 规范](/docs/simple-world/endpoints)构建，以确保每个前端和后端都能模块化组合
- 每个前端都使用同一套手工制作的 [Bootstrap 4 主题](https://github.com/gothinkster/conduit-bootstrap-template)，保证一致的 UI/UX
- 后端 API 提供了一个可公开使用的[托管版本](https://realworld-docs.netlify.app/docs/specs/frontend-specs/api#demo-api)，无需 API key
- 想创建一个新的 RealWorld 技术栈？查看我们的[后端 API 规范](/docs/simple-world/endpoints)


**通用功能：**

- 发表文章时只需要输入用户名
- 发表评论时只需要输入用户名
- 用户名为默认头像
- 对文章执行增删改查，任何人都可以编辑和删除文章
- 对文章评论执行增删查，
- 获取并展示分页文章列表
- 按标签筛选文章


## 请记住：让你的代码库保持_简单_，同时保持_健壮_。

如果一个刚接触你所用框架的新开发者，需要超过 10 分钟才能理解整体架构，那很可能说明你的工程设计有点过度了。

另一方面，你也_绝不能_为了简单而放弃基本最佳实践，否则就会把错误的做事方式教给同一个新手开发者。

Conduit 实现的质量和架构应该类似早期创业公司的 MVP：功能完整、稳定，但不要做不必要的过度工程化。


## 其他期望

- 应实现所有必需功能（见规范）。
- 你应该把实现发布到一个专门的 GitHub 仓库，并开启 “Issues” 区域。
- 你应该提供一份 README，介绍实现概览并说明如何在本地运行。
